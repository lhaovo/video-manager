# 视频管理工具需求文档

## 目标

用于管理从国内社媒收集、处理后再发布到国外多个社媒平台的视频。

第一版重点解决：

- 区分未处理和已处理的视频。
- 按视频时长筛选。
- 手动添加视频到视频库。
- 扫描指定视频文件夹并自动入库。
- 记录视频已经发布到哪些平台。
- 支持自定义发布平台。

视频文件本体保存在 NAS 上，系统只保存视频元数据、处理状态和发布记录。

## 第一版范围

### 视频管理

每个视频需要保存以下信息：

- ID
- 文件名
- 相对路径
- 所属库：未处理库 / 已处理库
- 文件大小
- 视频时长，单位为秒
- 处理状态：未处理 / 已处理
- 备注
- 添加时间
- 更新时间

暂时不保存：

- 来源平台
- 来源链接

### 处理状态

视频处理状态只保留两个：

- 未处理
- 已处理

处理状态和发布状态分开管理。

示例：

- 一个视频可以是“已处理”，但还没有发布到任何平台。
- 一个视频可以是“已处理”，并且已经发布到 TikTok、YouTube、Instagram。

### 发布平台

系统需要支持平台管理。

平台信息包括：

- ID
- 平台名称
- 是否启用
- 排序值
- 创建时间
- 更新时间

第一版可以预置常见平台，也必须支持手动新增、编辑、禁用。

可预置平台：

- TikTok
- YouTube
- Instagram
- Facebook
- X
- Pinterest

后续可扩展为“平台 + 账号”模型，但第一版只管理平台。

### 发布记录

一个视频可以发布到多个平台，一个平台也可以对应多个视频。

需要单独保存视频和平台的发布关系：

- ID
- 视频 ID
- 平台 ID
- 发布时间
- 备注

前端展示时可以显示为：

```text
已发布：TikTok, YouTube, Instagram
```

### 筛选能力

第一版支持以下筛选：

- 按处理状态筛选：
  - 全部
  - 未处理
  - 已处理
- 按时长筛选：
  - 全部
  - 小于 1 分钟
  - 1 到 5 分钟
  - 5 到 15 分钟
  - 15 分钟以上
- 按发布平台筛选：
  - 全部
  - 未发布
  - 已发布到指定平台
- 按文件名搜索。

### 添加视频

第一版支持三种入库方式：

- 手动添加到未处理库。
- 手动添加到已处理库。
- 扫描文件夹自动入库。

手动添加时：

- 用户选择或填写视频路径。
- 系统读取文件名、大小、时长。
- 用户选择处理状态。
- 系统保存到数据库。

### 自动扫描

第一版配置两个扫描目录：

- 未处理视频目录
- 已处理视频目录

扫描逻辑：

1. 遍历配置目录下的视频文件。
2. 判断数据库里是否已经存在该视频。
3. 不存在则读取文件名、大小、修改时间、视频时长。
4. 根据扫描目录设置所属库和处理状态。
5. 写入数据库。

支持的视频格式：

- `.mp4`
- `.mov`
- `.mkv`
- `.webm`
- `.avi`
- `.m4v`

第一版重复判断规则：

- 同一所属库下，相同相对路径的视频视为同一个视频。

后续如需要跨目录去重，再增加文件 hash。大视频计算 hash 成本较高，第一版不做。

### 视频库页面

视频库是第一版主页面。

需要包含：

- 状态筛选
- 时长筛选
- 发布平台筛选
- 文件名搜索
- 扫描按钮
- 添加视频入口
- 平台管理入口
- 视频列表

视频列表展示：

- 文件名
- 时长
- 文件大小
- 处理状态
- 已发布平台
- 添加时间
- 操作按钮

单个视频操作：

- 标记为未处理
- 标记为已处理
- 标记已发布到某个平台
- 取消某个平台的发布标记
- 删除入库记录

第一版不直接删除 NAS 上的视频文件，只删除数据库记录，避免误删。

## 暂不实现

第一版暂不实现：

- 来源平台和来源链接记录。
- 多账号发布管理。
- 自动上传国外社媒平台。
- 视频剪辑或转码。
- 团队权限。
- 多用户系统。
- 文件 hash 去重。
- 直接删除 NAS 源文件。
